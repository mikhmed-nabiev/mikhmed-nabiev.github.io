Bayesian Ensembling in Practice

Why Bayesian ensembling?

    link to project: https://github.com/intsystems/bensemble/tree/master

Imagine the usual story. You train a neat regression network, the validation curve behaves, the RMSE looks respectable. On the test set everything seems fine, so you feel safe. Then you move to slightly shifted data, or to some corner of the feature space that was barely covered during training, and suddenly the model is confidently wrong.

The issue is not only the prediction itself, but the lack of an honest “I don’t know.” A standard neural network gives a single parameter vector θ⋆\boldsymbol{\theta}^\starθ⋆ and a single prediction fθ⋆(x)f_{\boldsymbol{\theta}^\star}(\mathbf{x})fθ⋆​(x) for each input. All uncertainty is hidden inside early stopping heuristics, random seeds and architecture choices.

The Bayesian view starts from a simple observation: if there are many weight configurations that explain the data reasonably well, we should not pretend that one of them is the truth. Instead we keep a distribution over weights,

p(θ∣D)∝p(θ)p(D∣θ),p(\boldsymbol{\theta} \mid \mathcal{D}) \propto p(\boldsymbol{\theta})p(\mathcal{D} \mid \boldsymbol{\theta}), p(θ∣D)∝p(θ)p(D∣θ),

and make predictions by averaging over many plausible networks,

p(y∣x,D)=∫p(y∣x,θ)p(θ∣D)dθ.p(y \mid \mathbf{x}, \mathcal{D}) = \int p(y \mid \mathbf{x}, \boldsymbol{\theta}) p(\boldsymbol{\theta} \mid \mathcal{D}) d\boldsymbol{\theta}. p(y∣x,D)=∫p(y∣x,θ)p(θ∣D)dθ.

This already explains why Bayesian models tend to be more cautious off-distribution: if different networks in the posterior disagree, the averaged prediction becomes uncertain.

For deep networks this integral is intractable, so in practice we approximate the posterior with something tractable, call it q(θ)q(\boldsymbol{\theta})q(θ), draw samples

θ(1),…,θ(M)∼q(θ),\boldsymbol{\theta}^{(1)},\dots,\boldsymbol{\theta}^{(M)} \sim q(\boldsymbol{\theta}), θ(1),…,θ(M)∼q(θ),

and treat them as an ensemble. The core theme of this project is exactly that: build several approximate Bayesian neural networks, sample model ensembles from each of them, and compare their uncertainty behaviour in a controlled way.

The four methods we focus on are:

  • Probabilistic Backpropagation (PBP),
  • Laplace approximation,
  • variational Rényi inference,
  • a practical variational inference (PVI) scheme based on Bayesian linear layers.

They all target the same object p(y∣x,D)p(y \mid \mathbf{x}, \mathcal{D})p(y∣x,D), but they approximate p(θ∣D)p(\boldsymbol{\theta} \mid \mathcal{D})p(θ∣D) in very different ways.

A minimal Bayesian template

To keep the notation compact, let /(\mathcal{D} = (\mathbf{x}n, y_n){n=1}^N/) and a neural network /(f_{\mathbf{\theta}}(\mathbf{x})/) with parameters /(\mathbf{\theta}/). We assume

p(D∣θ)=∏n=1Np(yn∣xn,θ),p(\mathcal{D} \mid \boldsymbol{\theta}) = \prod_{n=1}^N p(y_n \mid \mathbf{x}_n, \boldsymbol{\theta}), p(D∣θ)=n=1∏N​p(yn​∣xn​,θ),

with a Gaussian likelihood for regression, and a prior p(θ)p(\boldsymbol{\theta})p(θ).

The exact posterior and predictive distribution are

p(θ∣D)∝p(θ)p(D∣θ),p(y∣x,D)=∫p(y∣x,θ)p(θ∣D),dθ.p(\boldsymbol{\theta} \mid \mathcal{D}) \propto p(\boldsymbol{\theta}) p(\mathcal{D} \mid \boldsymbol{\theta}), \qquad p(y \mid \mathbf{x}, \mathcal{D}) = \int p(y \mid \mathbf{x}, \boldsymbol{\theta}) p(\boldsymbol{\theta} \mid \mathcal{D}), d\boldsymbol{\theta}. p(θ∣D)∝p(θ)p(D∣θ),p(y∣x,D)=∫p(y∣x,θ)p(θ∣D),dθ.

All methods in this project fit into the same approximation pattern. We pick an approximate posterior q(θ)q(\boldsymbol{\theta})q(θ) and use Monte Carlo,

p(y∣x,D)≈1M∑m=1Mp(y∣x,θ(m)),θ(m)∼q(θ).p(y \mid \mathbf{x}, \mathcal{D}) \approx \frac{1}{M} \sum_{m=1}^M p\bigl(y \mid \mathbf{x}, \boldsymbol{\theta}^{(m)}\bigr), \qquad \boldsymbol{\theta}^{(m)} \sim q(\boldsymbol{\theta}). p(y∣x,D)≈M1​m=1∑M​p(y∣x,θ(m)),θ(m)∼q(θ).

The only question is how we define q(θ)q(\boldsymbol{\theta})q(θ) and how we learn it from data.

Probabilistic Backpropagation

Probabilistic Backpropagation (PBP) pushes the classical “uncertainty in weights” idea quite literally. Instead of a single value for each weight, PBP maintains a Gaussian

wi∼N(mi,vi),w_i \sim \mathcal{N}(m_i, v_i), wi​∼N(mi​,vi​),

and similarly treats noise and weight precisions as random, with Gamma posteriors. It is implemented for regression with a Gaussian likelihood.

Conceptually, for each input x\mathbf{x}x the network does not produce a single scalar fθ(x)f_{\boldsymbol{\theta}}(\mathbf{x})fθ​(x), but an approximate predictive distribution

p(y∣x,q)≈N(y∣μ(x),σ2(x)),p(y \mid \mathbf{x}, q) \approx \mathcal{N}\bigl(y \mid \mu(\mathbf{x}), \sigma^2(\mathbf{x})\bigr), p(y∣x,q)≈N(y∣μ(x),σ2(x)),

where μ(x)\mu(\mathbf{x})μ(x) and σ2(x)\sigma^2(\mathbf{x})σ2(x) arise from propagating means and variances through the network layers. Nonlinearities such as ReLU are handled by moment-matching: we approximate the output of ReLU under a Gaussian input by another Gaussian whose mean and variance are computed analytically.

Learning then proceeds in an assumed density filtering style. For each data point, we take the current approximate posterior qold(θ)q_{\text{old}}(\boldsymbol{\theta})qold​(θ), multiply it by the likelihood of that point, and project the result back to the Gaussian family:

qnew(θ)∝qold(θ)p(y∣x,θ)(projected back to Gaussians).q_{\text{new}}(\boldsymbol{\theta}) \propto q_{\text{old}}(\boldsymbol{\theta}) p(y \mid \mathbf{x}, \boldsymbol{\theta}) \quad \text{(projected back to Gaussians)}. qnew​(θ)∝qold​(θ)p(y∣x,θ)(projected back to Gaussians).

The end result is a factorized Gaussian over weights plus Gamma distributions over hyperparameters. From that, sampling full networks is straightforward: draw weights from the Gaussians, plug them into a standard multilayer perceptron, and you have a concrete ensemble member.

Laplace approximation

The Laplace approximation starts from a different angle. Instead of designing a Bayesian method from scratch, you begin with a network that has already been trained in the usual deterministic way, with weight decay capturing the prior. Let

ℓ(θ)=−log⁡p(θ∣D)=−log⁡p(θ)−log⁡p(D∣θ),\ell(\boldsymbol{\theta}) = -\log p(\boldsymbol{\theta} \mid \mathcal{D}) = -\log p(\boldsymbol{\theta}) - \log p(\mathcal{D} \mid \boldsymbol{\theta}), ℓ(θ)=−logp(θ∣D)=−logp(θ)−logp(D∣θ),

and let θ⋆\boldsymbol{\theta}^\starθ⋆ be the trained weights, approximately minimizing ℓ\ellℓ.

Laplace assumes that, near θ⋆\boldsymbol{\theta}^\starθ⋆, the posterior can be approximated by a Gaussian. Expanding ℓ\ellℓ in a second-order Taylor series gives

ℓ(θ)≈ℓ(θ⋆)12(θ−θ⋆)⊤H(θ−θ⋆),\ell(\boldsymbol{\theta}) \approx \ell(\boldsymbol{\theta}^\star)\tfrac{1}{2} (\boldsymbol{\theta} - \boldsymbol{\theta}^\star)^\top \mathbf{H} (\boldsymbol{\theta} - \boldsymbol{\theta}^\star), ℓ(θ)≈ℓ(θ⋆)21​(θ−θ⋆)⊤H(θ−θ⋆),

with H\mathbf{H}H a curvature matrix at θ⋆\boldsymbol{\theta}^\starθ⋆ (Hessian or Gauss–Newton). This corresponds to a Gaussian approximation

q(θ)=N(θ⋆,H−1).q(\boldsymbol{\theta}) = \mathcal{N}\bigl(\boldsymbol{\theta}^\star, \mathbf{H}^{-1}\bigr). q(θ)=N(θ⋆,H−1).

For modern networks, H\mathbf{H}H is far too large to store explicitly, so we use a Kronecker-factored approximation layer-wise. Roughly speaking, for a linear layer with input activations x\mathbf{x}x and pre-activation Hessian Hz\mathbf{H}_zHz​, the curvature block is approximated as

Hweights≈E[xx⊤]⊗E[Hz],\mathbf{H}_{\text{weights}} \approx \mathbb{E}[\mathbf{x} \mathbf{x}^\top] \otimes \mathbb{E}[\mathbf{H}_z], Hweights​≈E[xx⊤]⊗E[Hz​],

a Kronecker product of two much smaller matrices. This structure makes it feasible to invert the covariance and to sample weight matrices for each layer.

The attractive part is that no special training procedure is required: you can take any existing MLP, fit a Laplace approximation around its optimum, and immediately turn it into a Bayesian ensemble by drawing curvature-aware perturbations of its weights.

Variational Rényi inference

Variational Rényi inference keeps the spirit of standard variational inference but replaces the usual KL divergence by a whole family of divergences indexed by α\alphaα.

Rényi’s α\alphaα-divergence between ppp and qqq is

Dα(p∥q)=1α−1log⁡∫p(θ)αq(θ)1−αdθ.D_\alpha(p \Vert q) = \frac{1}{\alpha - 1} \log \int p(\boldsymbol{\theta})^\alpha q(\boldsymbol{\theta})^{1-\alpha} d\boldsymbol{\theta}. Dα​(p∥q)=α−11​log∫p(θ)αq(θ)1−αdθ.

When α→1\alpha \to 1α→1, this reduces to the familiar DKL(p∥q)D_{\mathrm{KL}}(p \Vert q)DKL​(p∥q). Different values of α\alphaα emphasize different aspects of the posterior: some are more mass-covering, others more mode-seeking.

In the implemented version, the approximate posterior q(θ)q(\boldsymbol{\theta})q(θ) is a factorized Gaussian over layer weights. The central quantity is the importance weight

w(θ)=p(θ,D)q(θ),w(\boldsymbol{\theta}) = \frac{p(\boldsymbol{\theta}, \mathcal{D})}{q(\boldsymbol{\theta})}, w(θ)=q(θ)p(θ,D)​,

and the Rényi-style objective

L∗α(q)=11−αlog⁡Eq(θ)[w(θ)1−α].\mathcal{L}*\alpha(q) = \frac{1}{1-\alpha} \log \mathbb{E}_{q(\boldsymbol{\theta})} \bigl[ w(\boldsymbol{\theta})^{1-\alpha} \bigr]. L∗α(q)=1−α1​logEq(θ)​[w(θ)1−α].

This is approximated by Monte Carlo using samples θ(k)\boldsymbol{\theta}^{(k)}θ(k) drawn via a reparameterization trick. When α=1\alpha = 1α=1, the bound collapses to the usual evidence lower bound (ELBO); for α≠1\alpha \neq 1α=1, we get a continuum of alternative objectives.

Qualitatively, this gives a knob that controls how aggressive or conservative the variational approximation is. Once trained, sampling networks is as simple as drawing from the Gaussian q(θ)q(\boldsymbol{\theta})q(θ) and plugging the sampled weights into the base model.

Practical variational inference

The practical variational inference (PVI) component in the project follows the “Bayesian layers’’ approach: instead of treating the whole network as a single variational object, it replaces ordinary linear layers by Bayesian linear layers and adds a simple Gaussian likelihood on top.

Each weight in a linear layer is modeled as

wij∼N(μij,σij2),w_{ij} \sim \mathcal{N}(\mu_{ij}, \sigma_{ij}^2), wij​∼N(μij​,σij2​),

with a fixed Gaussian prior p(w)p(\mathbf{w})p(w). Instead of sampling each weight independently at every step, PVI uses the local reparameterization trick: it samples directly in activation space. For an input mini-batch, the pre-activation means and variances are

γ=xW∗μ⊤,δ=x2W∗σ2⊤+bias term,\boldsymbol{\gamma} = \mathbf{x} \mathbf{W}*\mu^\top, \qquad \boldsymbol{\delta} = \mathbf{x}^2 \mathbf{W}*\sigma^2{}^\top + \text{bias term}, γ=xW∗μ⊤,δ=x2W∗σ2⊤+bias term,

and activations are drawn as

z=γ+ϵ⊙δ,ϵ∼N(0,I).\mathbf{z} = \boldsymbol{\gamma} + \boldsymbol{\epsilon} \odot \sqrt{\boldsymbol{\delta}}, \quad \boldsymbol{\epsilon} \sim \mathcal{N}(\mathbf{0}, \mathbf{I}). z=γ+ϵ⊙δ​,ϵ∼N(0,I).

The loss function is a standard variational one:

L=Eq(θ)[−log⁡p(D∣θ)]∗λKL,DKL(q(θ),∥,p(θ)),\mathcal{L} = \mathbb{E}_{q(\boldsymbol{\theta})}[-\log p(\mathcal{D} \mid \boldsymbol{\theta})] * \lambda_{\mathrm{KL}}, D_{\mathrm{KL}}\bigl(q(\boldsymbol{\theta}) ,\Vert, p(\boldsymbol{\theta})\bigr), L=Eq(θ)​[−logp(D∣θ)]∗λKL​,DKL​(q(θ),∥,p(θ)),

where p(D∣θ)p(\mathcal{D} \mid \boldsymbol{\theta})p(D∣θ) is a Gaussian likelihood with learnable noise scale and the KL term is a sum of closed-form divergences for each Bayesian linear layer. This is conceptually close to “Bayes by Backprop’’ but with a variance-reduction trick that makes training more stable.

After training, one can either keep sampling activations on the fly to get predictive distributions or freeze individual weight samples to obtain a more classical ensemble of deterministic networks.

Seeing the methods in action

The math above is intentionally light; the interesting part is how these methods actually behave on real data. Two notebooks in the project are central for that.

The first is demos, which focuses on library's interface and its capabilities.

The second is benchmark. This is the main playground where all four methods (PBP, Laplace, Rényi VI, PVI) are compared on the same dataset and with the same metrics against 3 baselines: a deterministic multilayer perceptron, an MC Dropout variant and a small HMC-based Bayesian linear model. Also we compared methods against an adversarial attack.

Conclusion

The project deliberately stays away from exotic new architectures. Instead it takes four representative approximations to Bayesian neural networks, puts them into a shared interface, and asks a simple question: if we look at their predictions as ensembles, how do they differ in terms of uncertainty?

Probabilistic Backpropagation uses moment propagation and ADF updates to maintain a Gaussian over weights and noise. Laplace starts from a deterministic optimum and wraps it with a curvature-based Gaussian. Variational Rényi inference plays with the divergence itself, using an α\alphaα-parameter to move between different approximations. Practical variational inference builds Bayesian behaviour into the layers and uses a classical variational loss with a KL penalty.

All of them aim at the same conceptual object: the Bayesian predictive distribution that averages over plausible parameter values instead of trusting a single network. The value of having them in a single codebase, with common notebooks and metrics, is that these differences stop being abstract and become visible: in calibration plots, in OOD behaviour, and in how fast confidence grows once the model leaves the comfort zone of the training data.
